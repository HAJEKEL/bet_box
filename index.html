Bet box

Leveraging React to construct a simple higher or lower game website with a component-based architectural approach, harnessing the virtual DOM and JSX for development.

Abstract

This study presents a web application implemented in React, a JavaScript library known for its component-based architecture, virtual DOM utilization, and JSX syntax. The application showcases a simplified betting game, offering insight into its development process and key components.

Introduction

In this scientific report, we detail the development of a web application using React, a JavaScript library renowned for its component-based structure, virtual DOM implementation, and use of JSX syntax. The application serves as a demonstrative case study, exemplifying the construction of a basic betting game.

Methodology

The application is structured as follows:

  1. main.tsx:
    • Entry point responsible for rendering the React application.
    • Imports the root component, App, and manages the rendering process.
  2. App.tsx:
    • The primary component serving as the foundation of the application.
    • Imports the Controller component, responsible for overseeing core functionalities.
    • Renders the Controller component within a containing div element.
  3. Controller.tsx:
    • Centralizes state management and logic for the betting game.
    • Utilizes React Hooks (useState and useEffect) to manage state variables, including hasWon, betDirection, isLoading, and valStored.
    • Defines a handlePlaceBet function to orchestrate the betting process when the "Place Bet" button is interacted with.
    • Initiates an external API request to retrieve a random number, subsequently determining the betting outcome.
    • Renders the Button component and the BetBox component, passing relevant state and functions as props.
  4. Button.tsx:
    • Represents the "Place Bet" button component.
    • Accepts a runFunction prop to execute a designated function upon user interaction.
    • Renders a button element with dynamic styling dependent on the selected betting direction.
  5. BetBox.tsx:
    • Manages the presentation of betting-related information.
    • Receives several props, including betDirection, setBetDirection, isLoading, valStored, and hasWon.
    • Displays the random number retrieved from the external API, the betting buttons, and the resultant game outcome.
  6. BetButton.tsx:
    • Represents individual directional betting buttons (up or down).
    • Receives props to dictate its behavior, encompassing direction, betDirection, and setBetDirection.
    • Renders a button component with conditional styling contingent on the selected direction.
